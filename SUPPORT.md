# Support

## Before opening an issue

1. Check [Compatibility and known limitations](docs/COMPATIBILITY.md).
2. Search existing GitHub issues for the same behavior.
3. Confirm the problem on the latest tagged release or current `main` branch.
4. Keep the original workbook private unless it contains no sensitive data.

## Bug reports

Use the GitHub bug report form and include:

- 900Sheets version
- Operating system and version
- Whether you used a release artifact or built from source
- Exact steps that reproduce the problem
- What you expected and what happened
- A small test workbook with invented data, when possible

Do not post customer lists, financial records, access tokens, passwords, or other private material.

## Questions and feature requests

Use GitHub Discussions if enabled. Otherwise, open a feature request. Explain the workflow you need, who it helps, and which file formats or spreadsheet applications must interoperate with it.

## Security reports

Do not use a public issue for a security vulnerability. Follow [SECURITY.md](SECURITY.md).

## Project scope

900Sheets is maintained as an open-source project. Support is best effort. Clear reproduction steps and small compatibility fixtures make issues much easier to investigate.
